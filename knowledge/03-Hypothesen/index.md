# Hypothesen

Diese User Stories sind **Hypothesen**, nicht validierte Anforderungen. Sie sollten mit Stakeholdern ueberprueft werden.

---

## Warum Hypothesen statt Anforderungen?

- Wir wissen noch nicht, welche Probleme Forster tatsächlich hat
- "Was brauchen Sie?" führt zu Wunschlisten, nicht zu Lösungen
- Rekonstruktive Fragen ("Was tun Sie konkret?") liefern bessere Erkenntnisse

## Annahmen über Forsters Arbeitssituation

| Annahme | Konfidenz | Validierungsfrage |
|---------|-----------|-------------------|
| Erstellt Berichte für Uni-Leitung | hoch | "Für wen erstellen Sie Berichte?" |
| Beantwortet Ad-hoc-Anfragen | hoch | "Wer fragt Sie nach Zahlen?" |
| Vergleicht VetMed mit Med Unis | mittel | "Mit wem vergleichen Sie sich?" |
| Arbeitet primär mit Excel | mittel | "Welche Tools nutzen Sie?" |
| Hat keinen Programmier-Hintergrund | hoch | "Haben Sie Erfahrung mit Code?" |

## Erfolgskriterien (kompetenzorientiert)

| Kriterium | Validierungsfrage |
|-----------|-------------------|
| Versteht den Datenfluss | "Können Sie erklären, woher ein Wert kommt?" |
| Kann Nutzen einschätzen | "Welche Ihrer Aufgaben erleichtert das Tool?" |
| Kann Grenzen benennen | "Was kann das Tool nicht?" |
| Versteht Promptotyping | "Könnten Sie einen ähnlichen Prozess starten?" |
